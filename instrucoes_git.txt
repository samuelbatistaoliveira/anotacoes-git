Curso Git e GIthub

Comandos

git init
inicializa um repositório Git em uma pasta que ainda não tinha controle de versão.

git status
Mostra o estado atual do repositório, indicando arquivos modificados, adicionados ou não rastreados.

.gitignore
Para ignorar uma pasta, coloque o nome dela no arquivo .gitignore:

nome_do_diretorio/


Para ignorar todos os arquivos Python:
Ignora todos os arquivos com extensão .py

git config user.name 'Samuel Batista' 
Define o nome do autor dos commits.

git config user.email 'samuelbbatista32@gmail.com'
Define o e-mail do autor dos commits.


Configuração global (vale para todos os repositórios):

git config --global user.name "Samuel Batista"
git config --global user.email "samuelbbatista32@gmail.com"

git add
Adiciona arquivos à staging area (área de preparação), marcando quais alterações serão incluídas no próximo commit.

git add .
Adiciona todas as alterações da pasta atual e de suas subpastas à staging area.

git commit -m "commit um"
Cria um commit com as alterações que foram adicionadas à staging area através do comando git add.

git commit --amend -m "Adicionando uma nova linha"
Altera a mensagem do último commit realizado.

git log
Exibe o histórico de commits do repositório, mostrando informações como autor, data, hash e mensagem de cada commit.

git config core.pager cat
Desabilita o pager e mostra tudo diretamente no terminal.

git config core.pager less
Habilita o pager "less", permitindo navegar pela saída de comandos longos.

git log -1
Exibe apenas o commit mais recente do repositório.

git log --oneline
Exibe o histórico de commits de forma resumida

git log --oneline -2
Exibe os 2 commits mais recentes de forma resumida

git log --before="2020-06-14"
Exibe os commits realizados antes da data informada.

git log --after="2020-06-13"
Exibe os commits realizados após a data informada.

git log --after="1 week ago"
Exibe os commits realizados na última semana.

git log --since="2 days ago"
Exibe os commits realizados nos últimos 2 dias.

git log --before="1 month ago" 
Exibe os commits realizados antes de um mês atrás.

git log --author="Samuel Batista"
Exibe apenas os commits realizados pelo autor informado.

git log --oneline --author="Samuel Batista"
Exibe de forma resumida apenas os commits realizados pelo autor informado.

git help log

git checkout <hash_do_commit>
Permite navegar para um commit específico do histórico do repositório.
Ao fazer isso, o Git entra no estado "detached HEAD", permitindo visualizar ou testar versões antigas do projeto.

git checkout master
para voltar para a branch principal

git switch master
retorna ao commit mais recente da branch master

git mv instrucoes_git.txt instrucoes1_git.txt
Move ou renomeia arquivos que estão sendo controlados pelo Git.

git rm instrucoes1_git.txt
Remove um arquivo do diretório de trabalho e também do controle de versão do Git.
A remoção ficará registrada no próximo commit.

git diff --staged
Exibe as diferenças entre o último commit (HEAD) e as alterações que estão na staging area (área de preparação).
É utilizado para revisar o que será incluído no próximo commit.

git restore --staged instrucoes_git.txt
Remove o arquivo da staging area (desfaz o git add), mas mantém as alterações no arquivo.

git checkout instrucoes_git.txt
Descarta as alterações feitas no arquivo e restaura a versão do último commit.

git reset HEAD --hard
Remove todas as alterações não commitadas e retorna o repositório exatamente ao estado do último commit.

"Um branch nada mais é do que uma ramificação do projeto."

branch master -> ramo principal

git branch
Lista todas as branches do repositório.
A branch atual aparece marcada com *.

git checkout nomedabranch
Troca para a branch informada.

git merge nome da branch a junção de dois branch
Mescla (junta) as alterações da branch informada na branch atual.

git rebase nomebranch
Reaplica os commits da branch atual sobre a branch informada, reorganizando o histórico.

git fetch
Baixa as atualizações do repositório remoto para o repositório local, mas não altera a branch atual nem os arquivos do projeto.

git pull
Baixa as atualizações do repositório remoto e as integra automaticamente na branch atual.

git push
Envia os commits do repositório local para o repositório remoto (GitHub, GitLab, etc.).

git init --bare
Cria um repositório Git sem diretório de trabalho (working directory).
É utilizado principalmente para criar repositórios remotos compartilhados em servidores.

git remote -v
Lista os repositórios remotos configurados no projeto, mostrando as URLs usadas para fetch e push.

git checkout -- .
Descarta todas as alterações não commitadas dos arquivos rastreados no diretório atual, restaurando-os para o estado do último commit.

git checkout -- nomedoArquivo
Descarta as alterações feitas em um arquivo específico e restaura a versão do último commit.

git push -u origin master
Envia os commits da branch master para o repositório remoto origin e define a branch remota como padrão para futuros push e pull.

git revert <commit id>
Cria um novo commit que desfaz as alterações de um commit anterior, sem remover o histórico.

git reset HEAD~1
Move o ponteiro HEAD um commit para trás, desfazendo o último commit local e mantendo as alterações nos arquivos.

git reset --hard HEAD
Remove todas as alterações não commitadas e retorna o repositório exatamente ao estado do último commit.
